## Specialist 6 — Agent safety & security auditor → `audit/agent-safety.md`

**Run mode.** Fable 5, code-only / headless. Every citation was re-opened and the quoted string confirmed; probabilistic items are marked Confidence: Possible/Probable with the reason.

**Headline (G1 holds everywhere).** The spine of this system is intact. All six advisory seams — synthesizer (`synthesis/claude.py`), QC-triage, pipeline-repair, node-author, feedback, archivist — expose **prose-only** JSON schemas (`_NARRATION_SCHEMA`, `_ADVICE_SCHEMA`, `_PROSE_SCHEMA`×2, `_THEMES_SCHEMA`, `_SUMMARY_SCHEMA`) with **no** verdict/confidence/finding/citation property the model can set. The verdict is always `aggregate_verdict(findings)` (`synthesis/base.py:28-32`), assembled deterministically; citations/`attach_to`/`scope`/ports/`addresses_rule_ids` come from the retriever+rules in the `_assemble_*` helpers, never the LLM. Every Claude path checks `response.stop_reason == "refusal"` **before** reading content and wraps the body in `try/except → stub`. Every env selector (`get_*_agent`, `get_synthesizer`) is `os.environ.get(..., "stub"/"jsonl").strip().lower()` then `== "claude"`, so any typo fails safe to the stub. `src/pipeguard/` shells out **nowhere** (`grep -rn 'subprocess|os.system|shutil.which' src/pipeguard/` → empty); only `api/routers/intake.py` + `api/routers/pipeline_run.py` invoke a subprocess, both via an argv list (no `shell=True`), with `run_id` regex-sanitized and `extra='forbid'` on the body. Node-author is core-only (`grep node_author api/` → empty; `grep propose_node frontend/src` → empty; only a `SettingsModelTier.tsx:53` roster row with `wired:false, phase2:true`). VAR-RTH-001 is disarmed by default (`RouteToHumanPolicy.significances == ()`), armed per-run via the `route_to_human` marker in `_active_runbook` (`main.py:228-243`), quotes `CLNSIG` verbatim (`rules.py:387`), and is rule-consumed, never sent to an LLM. The dry-run resolver refuses absolute/`..` locators before touching the FS (`pipelines_lifecycle.py:306-332`).

The findings below are the residual risks, all in the **egress de-identification** and **auth-posture** surfaces the maintainer flagged as HIPAA-critical. **No Blockers** (no agent-autonomy/verdict breach found).

### AS-01 · Live synthesizer egresses operator PII + subject identity to the Anthropic API · Medium · Confirmed · security-or-agent-autonomy-risk
- **Area / journey.** Decision-card narration (Operate → `/runs/:id`), live synthesizer path (`PIPEGUARD_SYNTHESIZER=claude`).
- **Evidence.** `src/pipeguard/synthesis/claude.py:87-102`, `_sample_context` builds the LLM `artifact_context` with `"metadata": meta.model_dump(exclude_none=True)` (:97) and `"log_excerpts": log_hits` (:93/:102). `Sample` (`src/pipeguard/models.py:309-316`) carries `subject_id`, `tissue`, `submitted_by`. So on the live path the synthesizer ships the subject key, tissue, and the operator name straight to the API.
- **Expected.** Least-privilege parity with the other agents: the archivist (`api/archivist.py:120-137,522-540`) and feedback agent (`api/feedback_agent.py:219-230`) send **only** de-identified aggregate counts; the export policy explicitly DROPs `submitted_by` ("operator PII has no business as an ML feature", `api/deid.py:155-176`).
- **Actual.** The live synthesizer sends `submitted_by` (which has zero narration value) plus `subject_id`/`tissue` verbatim to the LLM.
- **Root cause.** `_sample_context` dumps the whole `Sample` for chain-of-custody context without a field allow-list.
- **Min fix.** Drop `submitted_by` from the synthesizer metadata dump and pass only the fields the narration needs (or route the dump through `api.deid.redact`). Demo-critical: **N** (stub is the $0 default, G2); real de-id hole once live.
- **Risk of fixing now.** Low — narrows a payload; stub path unaffected. **Regression test.** Assert the `ClaudeSynthesizer` user payload contains no `submitted_by` and no raw `subject_id` for a sample carrying them.

### AS-02 · Share egress skips free-text redaction on the `headline`/`rationale` narration fields · Medium · Probable · security-or-agent-autonomy-risk
- **Area / journey.** `POST /api/runs/{id}/share` (Provenance → Share, D3 golden-path beat).
- **Evidence.** `api/safe_harbor.py:84-86` — `_FREETEXT_FIELDS = {"note","notes","comment","comments","narration","detail"}` (no `headline`/`rationale`). `api/main.py:976-981` builds the share row with `"headline": card.headline` and `"rationale": card.rationale`, then `redact_record(raw, origin)`. In `redact_record` those keys are not in `_DROP_FIELDS`/`_DATE_FIELDS`/`age`/`_FREETEXT_FIELDS`, so they hit the deid fallback (`:175`), which returns `PASSTHROUGH` for unnamed fields — emitted verbatim.
- **Expected.** Free-text egressing the boundary is scrubbed; the module's own guardrail (`safe_harbor.py:1-20`) claims mechanical identifier removal for "whatever leaves the machine."
- **Actual.** The card narration — which on the live synthesizer path (see AS-01) is grounded in a context containing `subject_id`/`tissue`/`submitted_by` — leaves with **no** redaction pass at all. (Note the scrub is weak anyway: `_LONGNUM_RE=\b\d{6,}\b` and no name pattern, so even a `SUBJ-0012` would slip — but here it is not even attempted.)
- **Root cause.** The narration fields were omitted from `_FREETEXT_FIELDS`.
- **Min fix.** Add `headline`, `rationale` (and `next_steps` items) to `_FREETEXT_FIELDS` so `redact_free_text` runs on them. Demo-critical: **N** (stub narration is templated from findings and does not echo identity values — confirmed `rules.py:133-165` embeds field *names*, not values).
- **Risk of fixing now.** Low. **Regression test.** Share a run whose `rationale` contains an email/phone/long-id and assert it is `[REDACTED:…]` in the emitted rows.

### AS-03 · RBAC gates trust a client-set role header and default to `approver` · Medium · Confirmed · security-or-agent-autonomy-risk
- **Area / journey.** All off-gate writes: `POST /api/runs/{id}/share` (`main.py:951`, `require_role("approver")`), `POST /api/runs` intake (`api/routers/intake.py:128`, `require_role("reviewer","approver")`), pipeline lifecycle, settings, review-queue.
- **Evidence.** `api/auth.py:50` `DEV_DEFAULT_ROLE: Role = "approver"`; `auth.py:115-130` `current_actor` reads the untrusted `X-PipeGuard-Role` header and, when absent, returns `Actor(id="dev", role="approver")`; frontend `api.ts:59-61` sets the header from client-side `RoleContext`.
- **Expected.** A stakes-y egress/exec write is authorized by a trustworthy principal.
- **Actual.** Any direct API caller self-asserts `approver` (or sends no header and *is* approver), so "approver-gated share" and "reviewer-gated intake" are advisory in the shipped config. Provenance UI copy says "approver-gated + confirm-gated" (`Provenance.tsx:76`), which slightly overstates the enforcement.
- **Root cause.** Intentional, **documented** dev shim (`auth.py:23-28,45-50`: "this default never ships as the production auth decision"). Flagged so the demo narration doesn't overstate the gate, not because it is a hidden defect.
- **Min fix.** For the demo, none required (offline/local, contrived data). For any shared deployment, swap `current_actor` for a verified identity provider (the single documented chokepoint) and default the header-less role to `viewer`. Demo-critical: **N**. **Regression test.** N/A for the shim; add a test asserting the header-less default role once a real IdP lands.

### AS-04 · `redact_record` leaks a guarded-origin cohort column when its value is `None` · Low · Confirmed · design-inconsistency
- **Area / journey.** `POST /api/runs/{id}/share` egress rows.
- **Evidence.** `api/safe_harbor.py:160-162` runs `if value is None: out[key] = None; continue` **before** the guarded-origin drop at `:177-179` (`if action is DeidAction.GATE_BY_ORIGIN and guarded: continue`). `tissue` is a `GATE_BY_ORIGIN` field (`deid.py:169-175`). The sibling module orders it the other way (`api/deid.py:129-131`: guarded-drop precedes the None check) and its docstring states the invariant explicitly (`deid.py:118-122`: "a GATE_BY_ORIGIN field on a guarded origin is still dropped even when None, so the mere presence of the column never leaks that a guarded run had a subject").
- **Actual.** For a guarded origin (`real-giab`/`unknown`) with `tissue` absent, the share emits `"tissue": null` instead of omitting the column, leaking that the run had (an empty) tissue slot — the exact schema-presence leak `deid.py` set out to prevent.
- **Min fix.** Move the guarded-origin `GATE_BY_ORIGIN` drop above the `value is None` short-circuit in `redact_record` (mirror `deid.redact`). Demo-critical: **N**.
- **Risk.** Low. **Regression test.** `redact_record({"tissue": None}, origin="real-giab")` must not contain the `tissue` key.

### AS-05 · `redact_record` deid fallback is case-sensitive while its field-name sets are case-insensitive · Low · Possible · design-inconsistency
- **Area / journey.** Share egress redaction.
- **Evidence.** `api/safe_harbor.py:158/163/172` match on `lkey = key.strip().lower()` (case-insensitive `_DROP_FIELDS`/`_DATE_FIELDS`/`_FREETEXT_FIELDS`), but the deid fallback at `:175` calls `base.action_for(key)` with the **original** `key`. `default_policy` names lowercase `subject_id`/`tissue` only (`deid.py:169-175`).
- **Actual.** A differently-cased cohort column (e.g. `"Tissue"`) would miss the `GATE_BY_ORIGIN` action and fall through to `PASSTHROUGH`, exporting it raw. Not triggered today (the share row keys are hardcoded lowercase at `main.py:972-980`), so this is latent.
- **Min fix.** Pass `lkey` to `action_for` (or lower-case inside `DeidPolicy.action_for`). Demo-critical: **N**. **Regression test.** `redact_record({"Tissue": "liver"}, origin="synthetic")` must pseudonymize, not passthrough.

### AS-07 · Synthesizer prompt-injection blast radius is bounded but non-zero on the live path · Low · Confirmed · post-hackathon-improvement
- **Area / journey.** Live synthesizer / triage narration.
- **Evidence.** `synthesis/claude.py:93` includes up to 8 `log_excerpts` filtered by `sample_id in ln`, plus full `finding.detail` (:114-117); triage sends `finding.detail`/`source_field` (`triage/agent.py:212-225`). A pipeline-authored `pipeline.log` line is untrusted text reaching the model.
- **Assessment.** Correctly **mitigated for the verdict**: `output_config.format` is a prose-only `json_schema`, and the verdict is `aggregate_verdict` (deterministic), so injected text cannot alter the gate. Residual risk is limited to *misleading advisory prose* on the live path (which the UI labels advisory). This is inherent to LLM narration; recording it as a post-hackathon hardening item (e.g. delimit/escape log excerpts), not a demo blocker. Demo-critical: **N**.

### AS-08 · Share manifest advertises all 18 Safe-Harbor classes though several are documented no-ops · Low · Possible · design-inconsistency

Consider labeling it "classes considered (some not present in this data model)". Demo-critical: **N**.
