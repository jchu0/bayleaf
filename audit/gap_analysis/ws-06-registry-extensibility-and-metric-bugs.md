# WS-06 — Registry-Driven Ingestion + Metric Correctness

## Problem
Adding one QC metric today requires editing four hardcoded sites in lockstep — the flat `QCMetrics` model (`models.py:320-347`, 13 named fields), the `_QCMETRICS_MAP` tuple (`metrics/mapping.py:23-40`), the column-by-column `parse_qc_metrics` (`parsers.py:210-237`), and a `QCThreshold` (`runbook.py:76-181`) — which is exactly why 7 of 20 registry keys are `# NOT COMPUTED`. `QCThreshold` is one-sided only (`runbook.py:13-42`), so `variant.titv`/`fold_enrichment` (registry `direction: target_band`) parse but can never score. And three real correctness bugs ride along: `dup_rate` unit scale (fixtures split between percent `11.0` and fraction `0.0057` under one fixed `percent` declaration → dup never gates), `mean_coverage` assay conflation, and a `% reads identified` mislabel.

## Design
Invert the ingestion contract from **named model fields** to a **registry-keyed observation map**. The rule engine already gates over a `dict[our_key → MetricValue]` (`rules.py:456`); the only thing hardcoded is the *source* of that dict. So:

1. **Registry-keyed ingestion (6b).** Introduce a per-sample `SampleMetrics{sample_id, raw: dict[our_key → RawObservation]}` as the ingestion contract. `parse_qc_metrics` iterates CSV columns, resolves each through `registry.resolve_alias(col)` (already exists, `registry.py:204-216`), and reads its scale from a new declared `raw_unit` on the registry source. `metric_values_for` becomes a generic loop calling `reg.observe(...)` over the map (no field enumeration). "Register a metric" = one YAML entry + one runbook threshold; ingestion picks it up for free. `QCMetrics` stays one release as a deprecated shim that lowers into `SampleMetrics` (back-compat + coordination window for WS-03).

2. **Target-band gate type (6c).** Add a `kind` to `QCThreshold` (`one_sided` default | `target_band`) plus band edges, and a `target_band` branch in `_evaluate_metric` (`rules.py:194-279`) that authors its own Evidence (`value; target [lo,hi]; hard [lo,hi]`). This lets `variant.titv` (~2.0–2.1 WGS / ~3.0 exome), `qc.fold_enrichment`, and `variant.allele_balance` score. It is also the mechanism WS-02 plugs its identity/contamination checks into without a 4-file change.

3. **Metric bugs (§9).** Fix the two fraction `dup_rate` fixtures to the driver's percent scale + regression test; relabel `mean_coverage`/`% reads` to what is actually computed and correct the registry source honesty; leave the assay-binding of the 30x coverage gate to WS-05's `RunbookSet`.

4. **Store consolidation (6d).** Collapse the seven `api/*_store.py` JSONL/SQLite/Postgres trios (~2,186 LOC, all the same "indexed columns + full-JSON document, env-selected, degrade-to-JSONL" shape — e.g. `share_store.py:1-60`) behind one generic `JsonlStore[ModelT]`; defer Postgres. Independent of the deterministic core; can land anytime.

## Risks / tradeoffs / honest limits
- **The pct-trap moves, it doesn't vanish.** Unit still can't be inferred from a column name; it must be *declared* — by the registry source (`raw_unit`) for the frozen CSV and by the WS-03 adapter for live tools. A mis-declared unit still mis-scales; the closed conversion table (`registry.py:113-119`) plus the required-unit guard keep it a loud error, not a silent 100x.
- **Deterministic-core surgery.** PR-1/2/3 touch the trust anchor; mitigated by making PR-1 verdict-neutral and asserting byte-identical `test_gate` output at each step.
- **`SampleMetrics` blast radius vs. the shim.** Keeping `QCMetrics` alive one release avoids a big-bang break but leaves two representations transiently; the shim is the coordination seam with WS-03 and must be deleted, not left to rot.
- **Registry single-source-per-key.** A `raw_unit` on one `MetricSource` can't express the same `our_key` arriving from two tools at two scales; the ingester-declares-unit rule (not the registry) is what actually resolves that — the registry `raw_unit` is only the frozen-CSV convenience default.
- **6d is scope, not correctness.** Store consolidation is real duplication but off the decision path; sequence it last and don't let it block the P1 metric fixes.

Each gap below ships as a **red test that can only go green when the real wiring exists**, plus a **standing anti-scaffold guard** that freezes the finding so it can't silently reopen.
