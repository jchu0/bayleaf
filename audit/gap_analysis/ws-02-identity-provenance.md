# WS-02 — Real Identity/Provenance Checks — System Fix Plan

## Problem
The "provenance & QC decision gate" has no working identity/contamination check: PROV-001 (`rules.py:43-82`) compares `SampleSheet.csv` against `demux_stats.csv` — but demux is *derived from* the sheet, so a real transcription typo propagates to both and is never caught; `DemuxRecord.reads`/`pct_reads` are parsed (`parsers.py:203-204`) but no rule reads them; and `contamination.freemix` / `identity.ngscheckmate_match` / `identity.sex_concordance` are registered-only, `# NOT COMPUTED` (`metric_registry.yaml:296-312, 262-294`) — no parser, no rule, invisible in the UI.

## Design
Four independent pieces, all keeping the verdict a deterministic function of cited `Finding`s (ADR-0001), each authoring its own `Evidence`, each pushing toward failing closed:

1. **Reframe PROV-001 as a wetlab/data-entry consistency check against a genuinely independent source.** Cross-check the sample sheet's `Sample_ID`/index against the accessioning/LIMS record (`sample_metadata.csv`, the Accession CRM — authored separately by the wetlab, `Accession.tsx`). Make index comparison **reverse-complement-aware** (i5 orientation flips by instrument workflow). Reframe the copy: "sample-sheet ⋈ accessioning consistency," reserve "chain of custody"/identity language for the genotype checks below.
2. **Real contamination detector — VerifyBamID2 FREEMIX end-to-end** on the existing dedup BAM: a driver command → `.selfSM` parser → `freemix` into `qc_metrics.csv` → a cited `CONTAM-001` rule. Un-flag `contamination.freemix` in the registry.
3. **Gate demux signals**: per-sample read-share dropout (`DEMUX-SHARE-001`, per sample) + run-level Undetermined-read fraction (`DEMUX-UNDET-001`, run scope) — the real signatures of an index problem / index hopping.
4. **Un-hide the identity/contamination catalog**: surface `contamination.freemix` (now computed), `identity.ngscheckmate_match`, `identity.sex_concordance` as first-class *expected* categories in the card readout, rendered **NOT RUN** when absent instead of silently omitted.

## Risks / tradeoffs / honest limits
- **VerifyBamID2 on a chr20-only panel** has few ancestry-informative markers → FREEMIX is low-power/noisy on this smoke-test panel. The *wiring* is real; the number's confidence isn't WGS-grade — label it as such in copy and docs. Full-genome/WES input is where it earns trust.
- **Reverse-complement permissiveness:** "equal OR revcomp" removes false positives from i5 orientation but opens a narrow false-negative window (a real i5 typo that happens to equal the revcomp of the correct index). More precise option: derive i5 orientation from `RunArtifacts.platform` (NovaSeq X/MiSeq forward vs 6000-v1.5 reverse) and compare in one orientation only. Recommend permissive default now, platform-driven orientation as a WS-05 follow-up.
- **freemix `required=False` is fail-open until WS-01 binds it** to the germline expected-set; interim honesty comes from the NOT_RUN readout row, but the *verdict* won't HOLD on absent freemix until WS-01 lands — call this out so the gap is deliberate, not accidental.
- **PROV-001 now depends on accessioned indexes.** Labs that don't record indexes at accessioning lose the check — surfaced as NOT RUN (honest), not silently passed; the demo fixtures must carry the indexes for the check to run.
- **Run-level findings via a reserved `"__run__"` key** avoid an engine/model refactor but introduce a card whose `sample_id` isn't a biological sample; the frontend should label it "Run-level (demultiplexing)". Alternative (a first-class run card) is a larger contract change deferred to WS-05.
