# W3 — Surface the downstream-of-variant-calling steps in Builder cards + Provenance

**Feature (one line).** Make the built-but-hidden post-variant-calling work honest and visible: fix the Lineage DAG lie where a variant stage reads "Not run" while VAR-RTH-001 fired an ESCALATE, add honest downstream provenance stages, and land the committed **RunDetail "QC Decision & Provenance Report" tab** (ADR-0018 §1.8 / D1) rendered over existing downstream data — with the Builder palette's dangling ports and card-only tools wired as the supporting slice.

**User value + persona/journey.** Persona: the **reviewer/approver** on the Operate flow reading a decided run — especially the golden-path CLINVAR-RTH ESCALATE beat (`AUDIT_PLAN.md:354`, "rules-decide moment"). Today the DAG (`/runs/:id/provenance`) *contradicts* the decision on the hero escalation, and there is no surface that terminates the run in a per-variant, human-signed report. Value: the run's downstream truth (route-to-human evidence, gate outcomes, what actually ran vs. "not run in this build") is coherent across the Lineage DAG, the Builder palette, and a real Report tab — and the E2E has a surface to terminate in (Queued deliverable #1).

## Approaches

### Approach A — Report-over-the-wire (smallest shippable slice; frontend-only)
**Sketch.** Two coupled frontend-only changes: (1) fix the "skipped-while-escalated" honesty bug in the Lineage `useMemo`; (2) render a Report view purely from `card.findings`/`.evidence`/`.gate_results`/`.next_steps` already on the wire, reusing the existing `CitedEvidence` table and the Provenance view-switch. The load-bearing insight: `VAR-RTH-001` already packages the verbatim CLNSIG + citation + `suggested_verdict=ESCALATE` into a `Finding` that crosses the wire inside `card.findings` — so **v1 needs no new `VariantCall` transport**.
- **Real files/seams (grounded).** Bug: `frontend/src/components/provenance/Lineage.tsx:94-116` — line **97** `if ((stage.key==='align'||stage.key==='variant') && arts.length===0) return 'skipped'` short-circuits *before* the gate-verdict check at `:98-102`; mirror note at `:130-132`. Render: `frontend/src/components/EvidenceTable.tsx:81` `CitedEvidence({findings, variant})` renders `e.value` verbatim (`:99,:142`), already consumed by `RunDetail.tsx:479,514,547`. Host: the Provenance three-view switch `Provenance.tsx:26-27` (`ProvView`, `VIEWS`) + `Tabs`. Data contract unchanged: `types.ts:25` (`Finding.evidence[]`), `:71` (`DecisionCard.findings`), `:70` (`next_steps`).
- **Reuses.** The `CitedEvidence` findings→evidence table (canonical verbatim renderer); the honest-stage vocabulary `STATUS_STYLE.skipped/partial` (`Lineage.tsx:29-70`); the Provenance view-switch pattern.
- **Pros.** Truly minimal, frontend-only, no migration/endpoint/fixture; fixes a Track-A truthfulness finding as a byproduct; G1-clean by construction (no write path).
- **Cons.** Builds no new downstream *capability*; caps richness at what became a `Finding` (benign/DP-only variants aren't in `card.findings`, so no full per-variant table); recommends the Provenance host, which sits against the maintainer's committed "RunDetail Report tab" wording.
- **Guardrail fit.** Strong — renders rule-authored findings, computes no verdict.

### Approach B — The complete build (new `RunReport` endpoint + RunDetail Report tab + Builder wiring)
**Sketch.** Three stacked, independently-shippable layers: (1) a **new `GET /api/runs/{id}/report`** serializing a `RunReport` that unions per-sample `VariantCall` rows (verbatim ClinVar) + route-to-human findings + gate outcomes + provenance pins — the load-bearing new backbone, because **no endpoint serializes `VariantCall` today** (confirmed: `api/report.py` does not exist); (2) a RunDetail **Report tab** ("QC Decision & Provenance Report", ADR-0018 D1) reusing the shipped `Tabs` primitive, the `RepairProposal` advisory prose+citation render shape, and the draft→approve + `useConfirm`/`Toast` sign-off; (3) honest downstream stages + the RTH bug fix + the Builder card/port-graph wiring (NGSCheckMate `ProcessSpec`, dangling reserved/optional ports).
- **Real files/seams (grounded).** `models.py:446-460` `VariantCall` (verbatim CLNSIG); `parsers.py:282` `parse_variant_calls`; new endpoint alongside `api/main.py:558+`; `RunDetail.tsx:11,204` (`Tabs`); `PipelineStage` `types.ts:127`, `STAGES` `Lineage.tsx:13-20`, `_ARTIFACT_STAGE` `api/main.py:558-582`; catalog `catalog.py:66-217` (7 specs; MultiQC `:200` inputs = `fastp_json`+`markdup_metrics`+`mosdepth_summary` only), NGSCheckMate card-only (`BuilderShared.tsx:220`), dangling reserved ports (`BuilderShared.tsx:323,338` `fastp_html`/`samtools_stats` reserved; `:345` `mosdepth_thresholds` required-but-dangling); compiler placeholder unchanged (`compiler.py:227-249`).
- **Reuses.** `Tabs`, `RepairProposal` advisory card, draft→approve lifecycle + `*_by` capture (ADR-0017), `ProcessSpec`+`Port` extension point, `DATA_EXPORTED` event template.
- **Pros.** Closes the whole committed gap; the `RunReport` read model is the natural home for the later gnomAD/`AnnotatedVariant` slice and the surface the acceptance E2E asserts against; directly answers maintainer refinement #1 (wire the full port graph).
- **Cons.** The new endpoint + `VariantCall` serialization + `types.ts` mirror is the single largest new surface in W3 and rides the hand-kept `types.ts` drift risk; multi-day; highest merge-collision surface (touches RunDetail, Lineage, catalog, BuilderShared concurrently with an active Builder session, G5); the full E2E depends on W1's run-path generality landing first.
- **Guardrail fit.** Strong if the endpoint stays read-only and the Report body is prose+verbatim-citations only, with a human-only sign-off (ADR-0018 L61).

### Approach C — Stage-manifest backbone (contract/data-model-first; root-causes the bug)
**Sketch.** A different axis: the reason the downstream is unsurfaced *and* the reason the RTH bug exists is one root cause — **a pipeline "stage" has no single source of truth**; it is re-derived independently in `types.ts:127` (`PipelineStage`), `Lineage.tsx:13-20` (`STAGES`), and `api/main.py:558-582` (`_ARTIFACT_STAGE`), and the frontend then *guesses* status from artifact-row counts (`Lineage.tsx:94-116`). Add a backend-owned `stages: RunStage[]` projection (`{key, ran, gate, gate_verdict, artifact_count, note}`) on `RunDetail`; Lineage, the Builder "runnable tier" labels, and the Report tab all *read* it instead of each re-deriving. The bug dies at the root (status is no longer inferred from artifact counts) and cannot be reintroduced.
- **Real files/seams (grounded).** Compute in `api/main.py` from `_artifact_stage_roles` (`:571`) + per-card `gate_results` (already on the wire, `RunDetail.tsx:204`); mirror `RunStage` into `types.ts:127`; repoint `Lineage.tsx:94-116`; `placeholder` runnable-tier backed by `compiler.py:226-249`; new downstream `ProcessSpec` would land in `catalog.py`; reserved-kind vocabulary `node_author/models.py:47,109`.
- **Reuses.** The hand-kept `types.ts` ↔ pydantic contract chain + read-only `GET /api/runs/{id}/…` projection shape; `Tabs` for the Report tab; `EventTrail` for share-as-event. No new store, agent, or abstraction.
- **Pros.** Fixes the honesty bug at the root, not as a one-off; one seam feeds all three W3 surfaces so they can't drift the way the current three do; Report v1 ships with zero new agent and zero new wire type (reads manifest + `card.findings`); "not run in this build" becomes a backend-computed truth, not a hardcoded frontend string.
- **Cons.** More upfront plumbing than a pure UI slice; adds a new (small) contract surface to keep in sync; does not by itself add the annotation `ProcessSpec` or a full per-variant table.
- **Guardrail fit.** Strong — the manifest carries a *copied* `gate_verdict` (lifted verbatim from the rule engine's `gate_results`), never an authored verdict/confidence.

## Recommended approach + why

**Merge all three into one 4-slice stack: ship A's honesty fix first, make it durable with C's stage manifest, land the committed RunDetail Report tab (B's host) rendered over A's existing-`Finding`s insight, with B's Builder/port-graph wiring as the refinement-#1 slice.** No single approach wins alone: A ships fastest but leaves stage truth fragmented (re-driftable) and hosts against the committed decision; B closes the whole gap but leads with the heavy new endpoint before proving the honesty model and carries the highest merge risk; C has the best root-cause instinct but over-plumbs slice 1 and defers the Report tab.

The graft:
- **Take A's smallest-slice discipline and its key insight** that the committed "MVP over EXISTING downstream data … no interpretation agent for v1" is satisfiable *without a new endpoint*, because VAR-RTH-001 already emits verbatim CLNSIG into `card.findings`.
- **Take C's stage manifest** as the durability layer, because the maintainer's own W3 grounding names the three-place fragmentation as the defect's source — the manifest makes the bug unreintroducible and backs the Builder "runnable tier."
- **Take B's host decision and its Builder/port-graph leg**: the maintainer's Resolved note is explicit — *BUILD a real **RunDetail** Report tab* — so I resolve A's host tension in favor of RunDetail (the honest downstream *stages* stay in the Provenance Lineage DAG; the Report tab holds the per-variant advisory panel + sign-off). B's new `RunReport` endpoint is demoted from "the backbone" to an **optional slice gated on one open question** (full per-variant table vs. decision-relevant findings), which de-risks the whole item.

Result: each slice is independently demo-safe; the heavy endpoint is opt-in; the flagship truthfulness bug clears on day one.

## Guardrail / ADR-0001 impact

- **G1 (rules decide / AI advises) — reinforced, not eroded.** The Report tab, the provenance stages, and the manifest are **locations, never verdicts**. `gate_verdict`/the ESCALATE are lifted verbatim from the rule engine's `gate_results`/`rules.py:404 suggested_verdict`; the surfaces *display* them. The manifest and Report have **no write path** except a human sign-off that sets `signed_by`, never a verdict — reusing draft→approve (ADR-0017) and honoring ADR-0018 L61 ("PipeGuard can never mark a report final on its own"). *A design that let the Report/stage/manifest set or annotate a verdict would be disqualified — this stack has no such path.*
- **compose ≠ execute — untouched.** Slice 4's NGSCheckMate spec is a **human-authored** `ProcessSpec`; an uncatalogued downstream card still compiles to the `exit 1` placeholder (`compiler.py:227-249`); no core (`src/pipeguard/`) shell-out is added. Any newly-wired reserved kind is a governed registry addition mirrored in `node_author/models.py:47`, never a fabrication.
- **G3/G4.** ClinVar CLNSIG rendered **verbatim** via `Evidence.value` (`rules.py` `value=hit.clinvar_significance`, `EvidenceTable.tsx:99,142`); no pathogenicity authored; **no "confidence"** — `DecisionCard.confidence` stays `null` (`types.ts:67`).
- **G6/G7.** Per-variant list paginated (reuse `Pager`); any sign-off/share is confirm-gated (`useConfirm`) + audited (`DATA_EXPORTED`), no un-gated cascade.
