# PipeGuard data

Origin labels for everything in this directory. Full strategy:
[docs/data/strategy.md](../docs/data/strategy.md).

Origin labels: `real-giab` (real GIAB HG002 + NIST truth) · `synthetic`
(programmatically perturbed **from real data** for labeled failure modes) ·
`contrived` (invented values, no real provenance — hand-authored *or*
machine-generated; realistic formats, not derived from real data).

| Dataset | Origin | Notes |
|---|---|---|
| `mock_run_01/` | `contrived` | Hand-authored session-1 demo. Realistic Illumina formats, **invented** values — barcodes, QC numbers, and the planted S4 barcode swap + S5 borderline QC. **Not GIAB.** To be superseded/augmented by real + synthetic bundles. |
| `mock_run_02/` | `contrived` | Machine-generated by the synthetic run generator ([`pipeguard.synthetic.generator`](../src/pipeguard/synthetic/generator.py), T-013). QC-heavy plate: S1–S2 clean (PROCEED), S3 low-Q30 / S4 high-dup (HOLD), S5 crashed align step (RERUN), S6 barcode swap (ESCALATE). Values invented, **not GIAB**. |
| `mock_run_03/` | `contrived` | Machine-generated by the same generator. Provenance-heavy plate: S1/S6 clean (PROCEED), S2 missing intake field / S3 low-coverage (HOLD), S4 absent-from-sheet (ESCALATE), S5 crashed align step (RERUN). Values invented, **not GIAB**. |
| `mock_run_scale_30/` | `contrived` | Machine-generated **scale showcase**, driven at volume by [`pipeguard.synthetic.scale`](../src/pipeguard/synthetic/scale.py) over the same generator. 30 samples with zero-padded, uniform-width IDs (`S01`..`S30`) and a deterministic CLEAN-dominant spread that spans **all four verdicts** (proceed 13 / hold 10 / escalate 5 / rerun 2) and all eight finding types. Flows through `run_gate_from_dir` with zero core changes. Values invented, **not GIAB**. |

`mock_run_0{2,3}` and `mock_run_scale_30` are **reproducible** — regenerate all committed
runs with `uv run python -m pipeguard.synthetic` (bare invocation, backward-compatible;
writes here). The CLI now has subcommands: `demo` (the default — every committed run,
including the scale run), `scale` (one large run), and `bulk` (many runs — see below).
`mock_run_01` is hand-authored and never regenerated.

Each generated run self-declares its origin **twice, always in agreement**: in-band via a
`pipeguard-synthetic` line in `pipeline.log`, and out-of-band via an `origin` marker file —
both derived from a single `ORIGIN_LABEL="contrived"` constant. That marker is the
generator's 6th file, so a generated run bundle is **6 files, not 5**. The pre-existing
`mock_run_02/03` `origin` markers were reconciled `synthetic` -> `contrived` to match: these
runs carry invented values, not real data perturbed into failure modes, which is exactly the
`contrived` definition. See [`docs/data/strategy.md`](../docs/data/strategy.md) for origin
labels (the definitions themselves are unchanged).

## `synthetic_bulk` — regenerated, never committed

Bulk volume (dozens of runs, to exercise the frontend's scale affordances — pagination,
filtering, large verdict mixes) is regenerated **on demand** into **`data/synthetic_bulk/`**,
which is **git-ignored** (see `.gitignore`). Its bytes never enter the repo: the one committed
scale run (`mock_run_scale_30/`) shows scale out of the box, and the rest is regenerable —

```bash
uv run python -m pipeguard.synthetic bulk --count 24 --samples 12   # -> data/synthetic_bulk/
```

Each bulk run is `contrived`, seeded, and reproducible from its `(count, samples, seed)`.

## `real-giab` — fetched, never committed

Real GIAB HG002 truth data (origin `real-giab`) lands in **`data/real-giab/`**,
which is **git-ignored** (see `.gitignore`). The bytes are never committed; the
reproducible record is the committed accession manifest + fetch script:

| Committed record | Fetched into `data/real-giab/` (git-ignored) |
|---|---|
| [`scripts/giab_hg002_manifest.json`](../scripts/giab_hg002_manifest.json) — exact GIAB v4.2.1 URLs + published md5s | HG002 GRCh38 v4.2.1 truth VCF (+`.tbi`), high-confidence BED, and (opt-in) a panel-region reads slice |
| [`scripts/fetch_giab_hg002.py`](../scripts/fetch_giab_hg002.py) — idempotent, checksum-verifying fetcher | — |

Run it (needs network; the reads slice also needs the separate genomics
toolchain — see [`scripts/README.md`](../scripts/README.md)):

```bash
python scripts/fetch_giab_hg002.py --dry-run       # print the plan, download nothing
python scripts/fetch_giab_hg002.py                 # fetch the small truth artifacts
```

Do **not** commit raw reads or large artifacts — commit accessions + a fetch
script instead (see `.gitignore` and the data strategy).
