# PipeGuard data

Origin labels for everything in this directory. Full strategy:
[docs/data/strategy.md](../docs/data/strategy.md).

Origin labels: `real-giab` (real GIAB HG002 + NIST truth) · `synthetic`
(programmatically perturbed for labeled failure modes) · `contrived`
(hand-authored demo — realistic format, invented values).

| Dataset | Origin | Notes |
|---|---|---|
| `mock_run_01/` | `contrived` | Hand-authored session-1 demo. Realistic Illumina formats, **invented** values — barcodes, QC numbers, and the planted S4 barcode swap + S5 borderline QC. **Not GIAB.** To be superseded/augmented by real + synthetic bundles. |
| `mock_run_02/` | `contrived` | Machine-generated by the synthetic run generator ([`pipeguard.synthetic.generator`](../src/pipeguard/synthetic/generator.py), T-013). QC-heavy plate: S1–S2 clean (PROCEED), S3 low-Q30 / S4 high-dup (HOLD), S5 crashed align step (RERUN), S6 barcode swap (ESCALATE). Values invented, **not GIAB**. |
| `mock_run_03/` | `contrived` | Machine-generated by the same generator. Provenance-heavy plate: S1/S6 clean (PROCEED), S2 missing intake field / S3 low-coverage (HOLD), S4 absent-from-sheet (ESCALATE), S5 crashed align step (RERUN). Values invented, **not GIAB**. |

`mock_run_0{2,3}` are **reproducible** — regenerate with
`uv run python -m pipeguard.synthetic.generator` (writes here); each run is
labelled in-band by a `pipeguard-synthetic` line in its `pipeline.log`. `mock_run_01`
is hand-authored and never regenerated. See
[`docs/data/strategy.md`](../docs/data/strategy.md) for origin labels.

Do **not** commit raw reads or large artifacts — commit accessions + a fetch
script instead (see `.gitignore` and the data strategy).
