## Discussion

**Working discipline (held every phase).** Each unit shipped as **build → verify green
(ruff/mypy/pytest) → adversarial read-only review → fix → re-verify**. The review pass
paid for itself every time it ran: it caught the still-live `sys.path` shim (Phase 0),
the barcode gate mislabeled QC instead of preflight (Phase 1), and a demo-breaking
Claude-path `TypeError` that 500'd the triage endpoint *before* its fallback (T-015).
Green tests alone hid all three.

**Phase 0 — tooling.** uv single-source (`pyproject` + `uv.lock`), strict mypy + ruff,
pre-commit/pre-push hook tiers, editable install (killed the PYTHONPATH shim), `make`
targets. 28→ green.

**Phase 1 — schema + provenance seam.** Evolved the decision records to `schemas.md`
additively (frozen, content-hashed `Finding`/`Evidence`; derived `gate`/`signature`;
`GateResult`; per-gate `gate_results`); `identifiers.py` (UUIDv7 + hashing); T-019
(confidence omitted until grounded). Built the event seam (ADR-0002): `AnalysisRun`
anchor + append-only `EventLedger` emitting a 16-event trail through `run_gate`.

**Parallel tracks (maintainer's call).** The frontend port and agent work don't block —
the agent is advisory/off the critical path, the frontend just consumes the API. Ran the
**QC-triage agent build in a background git worktree** while building the frontend in the
foreground; merged + reviewed on completion. Stub-first ($0), a curated knowledge corpus
+ stdlib retrieval, `BAYLEAF_TRIAGE_AGENT=claude` flip.

**Productionization (ADR-0014).** Pulled the FastAPI + React port into hackathon scope
(maintainer chose the *full* port over a lean hero-flow, accepting overrun risk).
`api/` = FastAPI read-API; `frontend/` = React + Vite + Tailwind; **Streamlit kept as the
green fallback**. Built hero-first: run overview → decision cards → triage (agent note in
the UI) → provenance trail — all verified in-browser against the live API.
