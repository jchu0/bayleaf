## Discussion

**Scout first.** A read-only fan-out mapped the shared pattern + each unbuilt agent's readiness.
The pattern is fully pinned by the two shipped agents (QC-triage in core, Feedback-triage in
`api/`): env switch `BAYLEAF_<X>_AGENT=stub|claude` + `_MODEL`, a deterministic Stub that is
**both default and fallback**, a Claude class (lazy `anthropic`, refusal‚Üístub, any-error‚Üístub),
an advisory record with `advisory=Literal[True]` and **no verdict/confidence**, deterministic
citations + LLM-prose-only. So each new agent is ~1 module + tests, offline-green.

**#2 Pipeline-repair ‚Äî the input was the real decision.** The maintainer chose the heavier path:
consume a **recurring issue signature** from the monitoring rollup (not a single failure card).
That rollup already exists (`/api/monitoring` counts `Finding.signature` across runs), so the
agent takes a `RecurringSignature`, retrieves a curated remediation corpus
(`knowledge/pipeline_repair.jsonl`, grounded in ADR-0008's taxonomy + the runbook, **no invented
thresholds**), and emits a cited `RepairProposal{summary, attach_to(stage), scope(gate)}`.
Crucially it is **on-demand** (`GET /api/monitoring/signatures/{signature}/repair`) ‚Äî the ADR-0008
**~3√ó auto-escalation trigger stays deferred**; building the agent doesn't require the trigger.
Opus-high tier (ADR-0012 ¬ß3a) ‚Äî cross-run failure reasoning is the rare/hard case the ADR
reserves the expensive model for.

**#3 Archivist ‚Äî organizational, least-privilege.** Mirrors the feedback agent: indexes/summarizes
already-decided runs from the projection ‚Üí an `ArchiveDigest` (digest + export manifest + cross-run
index). Its input deliberately drops `subject_id/tissue/submitted_by` (structural least-privilege);
it never opens/moves/deletes a file or relabels an origin, and carries no verdict. Haiku tier.

**#5 Node-authoring ‚Äî deferred (blocked).** Its safety argument rests on a rigid typed
`ToolNode`/`ArtifactKind` target + a palette-injection seam that **don't exist in code**, hosted in
a Pipeline Builder frontend that's mid-redesign. Building it now targets a moving substrate.

**Revised an earlier assumption on the T-026 folder move.** I had said I'd do the `triage/ ‚Üí
agents/` consolidation as part of #2. On reflection, T-026's own note says *"don't do it mid-dev
on other tasks (import-path churn)"* ‚Äî and the frontend redesign is in flight ‚Äî so #2 landed as a
sibling module (`pipeline_repair/`) and the move stays deferred for a quiet window.
